# Auto-Sprint Agent

An elizaOS Action that turns "we should build X" chat messages into real Linear tickets — automatically.

No more good ideas dying in a Discord thread. Someone says it, the agent catches it, a ticket exists.

**Status:** ✅ Core logic built and tested — creates real Linear tickets. Discord listener wiring is the next step (see Roadmap).

---

## What it does

1. Takes in a chat message
2. Checks if it matches trigger phrases like *"we should build..."* or *"someone should make..."*
3. If yes, calls the Linear GraphQL API and creates a real ticket
4. Replies back with a link to the new ticket

```
"we should build a dark mode toggle"
        ↓
✅ Created a Linear ticket: https://linear.app/khushiii/issue/AGE-1/we-should-build-a-dark-mode-toggle
```

That's a real ticket, created by this code, sitting in a real Linear workspace right now.

## Why

Most Eliza builds right now are memecoin bots. This is a "product-minded" build — proof that Eliza Actions can plug into real dev tools (Linear, Notion, whatever) and do useful work for a team, not just talk.

## Stack

- [elizaOS](https://docs.elizaos.ai) — agent framework (the `Action` primitive)
- [Linear GraphQL API](https://linear.app/developers) — ticket creation
- TypeScript + Bun, no extra SDK needed

## Setup

### 1. Install the elizaOS CLI

```bash
bun add -g elizaos@beta
```

### 2. Clone this repo

```bash
git clone https://github.com/khushikumari239/auto-sprint-agent
cd auto-sprint-agent/plugin-my-plugin
bun install
```

### 3. Add your Linear credentials

Create a `.env` file inside `plugin-my-plugin/`:

```
LINEAR_API_KEY=lin_api_your_key_here
LINEAR_TEAM_ID=your_team_id_here
```

Get your API key: Linear → **Settings → Account → Security & Access** → new key → give it "Create issues" permission.

Get your team ID:

```bash
curl -X POST https://api.linear.app/graphql \
  -H "Content-Type: application/json" \
  -H "Authorization: YOUR_API_KEY" \
  --data '{ "query": "{ teams { nodes { id name } } }" }' | python3 -m json.tool
```

### 4. Run the test script

```bash
bun --env-file=.env test-action.ts
```

You should see it detect the trigger phrase, hit Linear's API, and print back a real ticket URL. Go check your Linear team — the ticket will be there.

## How it works (the actual code)

The core is a single elizaOS `Action` in `src/actions/createLinearIssue.ts`:

- **`validate()`** — checks if the incoming message matches trigger phrases. This is what elizaOS calls to decide "should this Action even run for this message?"
- **`handler()`** — if validate() says yes, this calls Linear's GraphQL API to create the issue, then replies with the ticket link.

`test-action.ts` at the repo root calls these two functions directly with a fake message — no full chat server needed to prove the logic works.
